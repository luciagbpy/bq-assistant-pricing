Cambios aplicados: Se agregó el filtro por vertical Pharmacy y se cambió el agrupamiento de día a semana usando DATE_TRUNC(registered_date, WEEK).

      
SELECT
  DATE_TRUNC(o.registered_date, WEEK)   AS semana,
  o.country.country_name                AS country,
  CASE WHEN dp.is_darkstore=TRUE THEN 'DMart'
       WHEN o.business_type.business_type_name='Market' THEN 'Convenience'
       ELSE o.business_type.business_type_name END AS vertical,
  COUNT(DISTINCT o.order_id)            AS orders,
  ROUND(AVG(f.delivery_fee_user), 2)    AS avg_delivery_fee
FROM `peya-bi-tools-pro.il_core.fact_orders` AS o
LEFT JOIN `peya-datamarts-pro.dm_order_profitability.fact_order_profitability` AS f
  ON o.order_id = f.order_id
  AND f.registered_date BETWEEN DATE_SUB(DATE_TRUNC(CURRENT_DATE(), MONTH), INTERVAL 4 MONTH) AND CURRENT_DATE()
LEFT JOIN `peya-bi-tools-pro.il_core.dim_partner` AS dp
  ON o.restaurant.id = dp.partner_id
WHERE o.registered_date >= DATE_SUB(CURRENT_DATE(), INTERVAL 30 DAY)
  AND o.business_type.business_type_name = 'Pharmacy'
  AND f.order_status = 'CONFIRMED'
GROUP BY 1,2,3
ORDER BY semana DESC, orders DESC

explicación del query

      

        
¿Qué hace este query?

        
Calcula el delivery fee promedio y la cantidad de órdenes agrupado por país y vertical, para el último mes. Solo incluye órdenes logísticas confirmadas (excluye pickup y marketplace).

        
Tablas que usa:

        
    
          
  • fact_orders (o) — base principal con info de cada orden
    • 
          
  • profitability (f) — para obtener el delivery fee real pagado por el usuario
    • 
          
  • dim_partner (dp) — para clasificar la vertical correctamente
    • 
        

        
Filtros aplicados: último mes calendario · order_status = CONFIRMED · solo órdenes con logística propia

        
Resultado: una fila por país + vertical, ordenada de mayor a menor por cantidad de órdenes.
